2007年07月22日

右クリック

　私は今、生まれて初めて、MacはWindowsを打ち負かすかもしれない、と思いはじめている。
　
http://forums.microsoft.com/MSDN/ShowPost.aspx?PostID=1529091&SiteID=1
If we add "right mouse events" in some future version it would probably be in the form of a "Context Menu" event only. Something that would fire however a context menu is opened.
　
http://forums.microsoft.com/MSDN/ShowPost.aspx?PostID=1096108&SiteID=1
Right now we do not have plans to expose right mouse button events.
　
http://silverlight.net/forums/t/431.aspx
I don't think there's going to be a "onRightMouseDown, 'move, 'enter, 'leave and 'up" set of events.
　
　Silverlight開発者のあいだで、これほど当惑の声が上がっているにもかかわらず、SilverlightのプログラムマネージャMark Rideoutは、頑として方針を変えようとしない。Silverlightは右クリックに対応しない――Flashと同程度にしか――という方針を変えようとしない。
　Mark Rideoutはあれこれ説明しているが、実際の理由は2つしかありえない。
1. Flashがそうしているから。
2. Macが1ボタンだから。
　AdobeとAppleに盲従するこの卑屈さは、市場の支配者にはふさわしくない。いや、支配者どころか、商売にふさわしくない。なぜ自分の強みを生かそうとしないのか？　Windowsのマウスの右ボタンはいつから盲腸に変わってしまったのか？
　Steve Jobsは絶対にこんな過ちを犯さないだろう。マウスのボタンが1個しかないこと（明らかに欠点）を、逆に強みにしてしまうべきだとJobsは知っている。また、そのために何をすべきかを知っている。Mark Rideoutはその逆だ。マウスのボタンが2個あること（明らかに長所）を弱みだと思い込み、不適切な前例を用いてその思い込みを補強している。
　Microsoftはその精神において壊れつつあるのかもしれないと、私は今、思い始めている。
